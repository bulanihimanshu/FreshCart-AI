# User and Authentication Endpoints

## Context
Hardcode 5 demo users and provide session-less basic auth for demonstration. The user tier (1, 2, or 3) should be computed or hardcoded based on `order_count`.

## Verification criteria
- [ ] `POST /api/auth/login` works with correct credentials, returns 401 otherwise
- [ ] must_haves:
  - Users are validated against local `users.json`
  - Route accurately computes/uses tier

## Task 2: Auth Endpoints
<read_first>
- backend/data/users.json
- backend/main.py
</read_first>

<action>
Create `backend/api/auth.py`.
Implement `router = APIRouter()`.
Implement `POST /api/auth/login` accepting `username` and `password` on JSON body.
Validate against `backend/data/users.json`. Return `{ user_id, display_name, order_count, tier }` on success, or HTTP 401 on failure. Calculate `tier` dynamically depending on `order_count` (0=tier1, 1-2=tier2, 3+=tier3) or just read it from the JSON.
Implement `POST /api/auth/logout` that just returns `{ "success": True }`.
Include router in `backend/main.py`.
</action>

<acceptance_criteria>
- `backend/api/auth.py` contains `@router.post("/api/auth/login")` and `@router.post("/api/auth/logout")`
- Login raises `HTTPException(status_code=401)` on failure
- `backend/main.py` has `app.include_router(auth.router)`
</acceptance_criteria>
