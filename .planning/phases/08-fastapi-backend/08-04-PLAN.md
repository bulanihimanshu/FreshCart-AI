# Recommendation Endpoint & Cold Start Routing

## Context
The core recommendation logic routing: Tier 1 (global top), Tier 2 (aisle affinity based on recent history), Tier 3 (LSTM + time).

## Verification criteria
- [ ] Recommendations correctly returned dynamically hydrated from products table
- [ ] Deduplication removes any item that is already in `cart_items`
- [ ] Always pad the result to exactly 5 items randomly (or sequentially) using Tier 1 backfills.
- [ ] must_haves:
  - Correct model inferences generated from Keras using current timestamp data

## Task 1: Cold Start Logic Router
<read_first>
- .planning/phases/08-fastapi-backend/08-CONTEXT.md
- backend/main.py (for app state)
</read_first>

<action>
Create `backend/core/cold_start.py`.
Implement a `get_recommendations(user, cart_items, hour, dow, days_gap, app_state)` function.
If `tier == 1`: Recommend items from `app.state.global_top20`.
If `tier == 2`: Use the last item in `cart_items` (or user's order history) to identify an aisle and return from `app.state.aisle_top10`.
If `tier == 3`: Format the `cart_items`, `hour`, `dow`, and `days_gap` into correct numpy arrays. Run `app.state.lstm_time_model.predict([seq, hour, dow, days])`. Fetch the top predicted classes using numpy `argsort`.

For all tiers, take the integer product ids and:
1. Deduplicate items already in `cart_items` (D-06).
2. Grab the top 5 product IDs.
3. Fallback/padding: If the resulting list is less than 5 items, pad with items from `app_state.global_top20` that aren't in the cart (D-07).
4. Hydrate item details from `app.state.products_df` using the IDs mapping (D-05).
Return the list of `{product_id, name, aisle, department}`.
</action>
